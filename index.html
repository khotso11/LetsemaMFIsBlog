Introduction

The Letsema project is a distributed microfinance management system supporting Borrowers, MFI Admins, and Letsema Admins. This report outlines its development, challenges, and current system state.

Database Design

Key tables and fields include:

  • Loan: loan_id, user_id, mfi_id, amount, interest_rate, etc.
  • User: user_id, first_name, last_name, email, role, etc.
  • MFI: mfi_id, name, location, contacts.
  • Loan Repayment: repayment_id, loan_id, amount_paid, payment_date.
  • MongoDB: Used for borrower credit histories.

Distributed Database Architecture

Originally attempted with Citus, the system moved to a simpler Master–Master PostgreSQL replication approach, integrated with Django multi-tenancy and MongoDB for non-relational data.

Each MFI operates under its own subdomain: [tenant].localhost:8000

Frontend Development

The frontend, built with React.js, currently supports basic admin interfaces. Borrower dashboards, loan application UI, and repayment views are under development.
