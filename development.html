Development Journey

Challenges and Solutions

Initial Challenges: The original top-down database design tightly coupled MFIs with the core schema. Adding new MFIs required code and schema changes. Borrower self-registration was missing, affecting UX negatively.

Solutions Implemented:

  • Shifted to bottom-up, multi-tenant schema separation.
  • Implemented automated MFI onboarding APIs and dynamic schema creation.
  • Added borrower self-registration and mobile-first UX improvements.

Frontend Challenges

React.js frontend faced delays due to API changes and lack of frontend expertise. Borrower dashboards, loan application UIs, and repayment views are still under construction.

Next Steps

  • Standardize API response formats.
  • Complete borrower and repayment UI flows.
  • Expand frontend component modularity.
